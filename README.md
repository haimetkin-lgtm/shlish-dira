# שליש דירה — זירת ביתא

דף נחיתה וזירה סגורה למודל "מכירת שליש דירה" כאלטרנטיבה למשכנתא הפוכה:
מחשבון השוואה אינטראקטיבי, רישום מוכרים, משקיעים ועורכי דין מייצגים, והזמנת שמאות.

## הרצה מקומית

```bash
npm install
npm run dev
```

## חיבור Supabase (טפסי הרשמה)

1. פרויקט חדש ב-Supabase, להריץ את `supabase/schema.sql` ב-SQL Editor.
2. להעתיק `.env.local.example` ל-`.env.local` ולמלא URL + anon key.
3. הטבלה insert-only מהדפדפן: קוראים לידים רק מהדשבורד של Supabase.

בלי הגדרת Supabase האתר עולה רגיל, והטפסים מציגים הודעה שההרשמה עוד לא פעילה.

## פריסה ל-GitHub Pages עם הדומיין viager.co.il

1. ריפו `shlish-dira`, push ל-main מפעיל את `.github/workflows/deploy.yml`.
2. Settings > Pages > Source: GitHub Actions. בשדה Custom domain: `viager.co.il` (קובץ `public/CNAME` כבר קיים).
3. Settings > Secrets and variables > Actions: להוסיף `NEXT_PUBLIC_SUPABASE_URL` ו-`NEXT_PUBLIC_SUPABASE_ANON_KEY`.
4. אצל רשם הדומיין של viager.co.il להגדיר DNS:
   - רשומות A ל-apex (`@`): `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
   - רשומת CNAME ל-`www`: `<username>.github.io` (להחליף בשם המשתמש בגיטהאב)
5. אחרי שה-DNS מתעדכן: לסמן Enforce HTTPS בהגדרות Pages.

הערה: מכיוון שהאתר מוגש בדומיין ייעודי, אין `basePath` ב-`next.config.ts`.

## לפני השקה — משימות פתוחות

- [ ] אימות מיסוי עם עו"ד: מס רכישה לרוכש שליש, מס שבח למוכר חלק (חישוב ליניארי מוטב), גבולות הטבת "עד שליש".
- [ ] בדיקת חוק המתווכים: האם גביית תשלום על חשיפת צדדים דורשת רישיון תיווך. עד אז ההכנסה מהשמאות בלבד.
- [ ] תבנית חוזית: זכות מגורים רשומה + הסכם שיתוף + מנגנוני פירוק שיתוף (סעיף 37 לחוק המקרקעין מגביל תניית אי-פירוק לשלוש שנים).
- [ ] חיבור Cardcom לתשלום על שומה וחשיפת פרטים (דף redirect עם UUID, כמו בפרויקטים קודמים).
- [ ] שלב ב: אזור זירה עם כרטיסי נכס אנונימיים (עיר, טווח שווי, קבוצת גיל, תשואה אקטוארית) וחשיפה בתשלום.
